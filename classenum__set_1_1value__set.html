Represents a set of values from a fixed non-empty universe of Values.... The class adapts a type_set with value semantics by decorating all methods to accept values instead of types (which then maps a Value of Type to value<Type, Value>). Documentation for all methods can be found in type_set.hpp, just replace any occurence of a type with value and it should be valid for this class as well.

Function documentation

template<typename Type, Type... Values> template<typename... Types>
enum_set::value_set<Type, Values>::value_set(Types... values) constexpr

Constructs a value set from a set of values. Providing an invalid value throws an std::out_of_range exception. Complexity is bilinear in the number of Values... of the value_set and the number of argument Types... provided in the constructor call, hence, if you use many Values..., use this constructor with care. If you know the values at compile time, use make factory method instead.

template<typename Type, Type... Values>
iterator enum_set::value_set<Type, Values>::begin() const constexpr noexcept

Returns an iterator to the first element in the value set. The iterator does not point to any underlying value, it only acts as an indicator of the existence of a certain element in the value set. See value_set_iterator class for details.

template<typename Type, Type... Values>
iterator enum_set::value_set<Type, Values>::end() const constexpr noexcept

Returns a sentinel iterator representing the end of this value set. See begin() for details.
